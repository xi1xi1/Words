## 心得体会

通过本次作业，我深入理解了：

1. **RESTful API 设计原则**：使用名词复数作为资源路径，通过 HTTP 方法表达操作意图，合理使用状态码

2. **Spring Boot 分层架构**：Controller 层只处理请求响应，Service 层处理业务逻辑，职责清晰

3. **统一返回格式的重要性**：`ApiResponse<T>` 让前后端协作更加规范，错误处理更加统一

4. **单元测试的价值**：编写测试用例能提前发现接口问题，保证代码质量
