## 遇到的问题和解决

### 问题1：首页信息层级过多，如何平衡功能完整性和视觉简洁性？

**解决**：参考“不背单词”的设计思路，将核心功能（学习/复习）用两大卡片突出展示，次要功能（今日推荐单词）放在下方，舍弃词库切换和学习动态，让首页聚焦于“立即学习”这一核心任务。

---

### 问题2：闯关模块内部页面跳转关系复杂，如何避免用户迷失？

**解决**：梳理出清晰的页面跳转流程图：
```
场次选择 → 闯关答题 → 结果页面 → 再玩一局/查看排行/返回主页面
```

---

### 问题3：学习数据仪表盘信息量大，如何让用户快速理解？

**解决**：采用“核心指标卡片 + 趋势图 + 热力图”的三段式布局，核心指标用大数字展示，趋势图用柱状图呈现变化，热力图用颜色深浅代表学习强度，符合用户阅读习惯。

## 四、心得体会

通过本次“背了么”App 的设计实践，我深刻体会到**信息架构是产品设计的基石**。一个好的架构不是简单罗列功能，而是要站在用户角度思考：用户最常用什么？用户的使用路径是怎样的？如何让用户最短路径完成任务？

在设计过程中，我学会了如何将抽象的产品需求转化为具体的功能模块，再组织成清晰的页面层级。特别是首页的设计，经历了多次调整——从最初有词库切换、学习动态，到最后聚焦于“学习/复习”两大核心任务，让我明白了“少即是多”的设计原则。
